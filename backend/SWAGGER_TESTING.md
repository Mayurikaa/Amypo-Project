# Swagger API Testing Guide — TaskGuard backend

This document shows step-by-step instructions to run the backend locally and test the API using Swagger UI and curl.

## Prerequisites
- Java (version compatible with the project) and Maven wrapper present in the repo.
- Port 8080 free (default Spring Boot port).
- A terminal (PowerShell or Bash).

## 1) Run the application

Windows (PowerShell):

```powershell
cd "c:\Users\Mayur\Downloads\Amypo Project\backend"
.\mvnw.cmd spring-boot:run
```

Linux / macOS:

```bash
cd "backend"
./mvnw spring-boot:run
```

Wait until Spring Boot finishes startup and logs "Started".

## 2) Open Swagger UI
- Open one of these in your browser:
  - http://localhost:8080/swagger-ui.html
  - http://localhost:8080/swagger-ui/index.html

You should see the TaskGuard API documentation generated by springdoc-openapi. The API is configured with a `BearerAuth` security scheme (JWT).

## 3) Obtain a JWT (login)
The app seeds a director account if not present:
- email: `director@gmail.com`
- password: `password`

Use the login endpoint to get a token:

Bash / curl:

```bash
curl -s -X POST "http://localhost:8080/api/v1/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"email":"director@gmail.com","password":"password"}'
```

PowerShell (Invoke-RestMethod):

```powershell
Invoke-RestMethod -Method Post -Uri "http://localhost:8080/api/v1/auth/login" -ContentType "application/json" -Body '{"email":"director@gmail.com","password":"password"}'
```

The response JSON includes a `token` field. Copy that JWT value.

## 4) Authorize Swagger UI with the token
- In Swagger UI click the "Authorize" button.
- The scheme name is `BearerAuth`.
- Enter the value using the `Bearer` prefix, e.g.: `Bearer eyJhbGci...` (paste the token after `Bearer `).
- Click "Authorize" then close the dialog.

Now Swagger will include the `Authorization: Bearer <token>` header on protected requests.

## 5) Try endpoints in Swagger UI
- Expand an endpoint (for example, `GET /api/v1/tasks/`), click "Try it out", then "Execute" to run the request.
- For protected endpoints (e.g., `POST /api/v1/tasks`), ensure you've authorized first.

### Example: List tasks (public)
- `GET /api/v1/tasks/` — use query params like `page` and `size` if desired.

### Example: Create a task (requires role PROJECT_DIRECTOR or PROJECT_MANAGER)
`POST /api/v1/tasks` body example (JSON):

```json
{
  "taskCode": "TASK-1001",
  "initiativeId": 1,
  "title": "Example task from Swagger",
  "description": "Short description",
  "priority": "HIGH",
  "estimatedHours": 8,
  "dueDate": "2026-07-31"
}
```

Click "Try it out" then "Execute" in Swagger UI.

## 6) Use curl with the JWT
After you have the token (replace `<TOKEN>` with the token string):

List tasks (curl):

```bash
curl -X GET "http://localhost:8080/api/v1/tasks/" \
  -H "Authorization: Bearer <TOKEN>"
```

Create a task (curl):

```bash
curl -X POST "http://localhost:8080/api/v1/tasks" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <TOKEN>" \
  -d '{"taskCode":"TASK-1002","initiativeId":1,"title":"CLI task","priority":"MEDIUM","estimatedHours":4,"dueDate":"2026-08-15"}'
```

## 7) Troubleshooting
- 401 Unauthorized: check you pasted the token with the `Bearer ` prefix in Swagger, or provided correct header in curl.
- 400 Bad Request: check required DTO fields and JSON shape; DTO validation messages are returned in the response.
- Port conflicts: change server.port in `src/main/resources/application.properties` or free port 8080.

## 8) Quick tips
- You can register a new user with `POST /api/v1/auth/register` (body matches `SystemAccountDto`).
- Use Swagger UI to view request/response DTO schemas for each endpoint.
